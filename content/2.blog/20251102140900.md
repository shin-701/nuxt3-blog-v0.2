久しぶりの投稿になります。
社会人も7年目になり、チームリーダーとして案件を進めることも出てきました。
そうしたときに出てくるのが、WBS。案件を遂行するにあたり、実施するタスクを細分化、一覧化したものです。
色々と試行錯誤していい感じのものができたので、ブログで残しておこうと思います。

## これまでのWBS管理の悩み

今までWBSを作成すること、チームで使うことは多々ありましたが、以下のような問題を感じていました。

### 1. タスクの期日や締切を可視化できてない

AsanaなどのTODO管理ツールでは、期日切れのタスクが分かりやすく表示されますが、スプレッドシートだとデフォルト機能ではうまく表示できません。

### 2. 今実施しているタスクがバラバラに散らばっていて見づらい

WBSは基本的に各工程ごとに上からタスクを切っていきますが、複数人で案件を進めていくと、「Aさんには先行してこの作業を進めてほしい」という状況が発生します。
夕会では、それぞれのメンバーが実施しているタスクを表示したいですが、スクロールせずに1画面で表示することが難しい場合もあります。


## 今回作成したWBSスプレッドシートの基本設計

今回作成したWBSスプレッドシートは、上記の問題を解決するために以下のようなカラム構成としました。

![image](https://my-obsidian-images-shin701.s3.us-east-1.amazonaws.com/2025/11/02/3c6d846ebfead2024445f61aac604853.png)

| カラム名      | 説明                                                    |
| ------------ | ------------------------------------------------------ |
| 工程                             | 要件定義、システム設計、実装／単体試験、内部結合試験、外部結合試験、総合試験などのタスクの工程を記載 |
| アラート                         | 関数で表示する項目<br>期限切れ：完了予定日が来ても完了してないタスク<br>着手遅れ：開始予定日が来ても着手してないタスク<br>期日まで：○日：期日までの残り日数(5日前から表示する) |
| 現在の担当者                      | 作業者とレビュー者どっちが作業中なのかを関数で表示|
| タスク                           | タスクを表示する欄 |
| タスクの詳細／メモ<br>※自由に記載可  | タスク関連メモを記載する欄。各タスクの担当者が色々メモしてほしい |
| 管理.予定FROM                    | タスクの開始予定日 |
| 管理.予定TO.                     | タスクの完了予定日 |
| 管理.分析.予定工数(人日)           | 管理.予定FROM - 管理.予定TOを関数で計算して表示 |
| 管理.分析.実績工数(人日)           | 実績を関数で計算して表示 |
| 作業.ステータス                   | 作業のステータス |
| 作業.担当者                      | 作業担当者 |
| 作業.実績FROM                    | 作業の実績開始日 |
| 作業.実績TO                      | 作業の実績完了日 |
| レビュー.ステータス                | レビューのステータス |
| レビュー.担当者                   | 作業担当者 |
| レビュー.実績FROM                 | 作業の実績開始日 |
| レビュー.実績TO                   | 作業の実績完了日 |

ポイントは実作業の【ステータス・担当者・開始日・完了日】とレビューの【ステータス・担当者・開始日・完了日】を分けたことです。
仕事で発生するほとんどのタスクには、上司のレビューが存在します。
レビュー中、作業担当者は別タスクの作業をすることができますが、ここをうまく管理するために管理カラムを分けました。
こうすることで、「レビュー中に作業者が別タスクを進めていること」をWBSでうまく表現することができます。

## 条件付き書式とグループ化表示

ステータスを判定することで今このタスクがどういう状況なのか、視覚的に分かりやすくしています。

例えば、以下のように条件付き書式を設定しています。
1. タスクが完了した場合：グレー背景
2. 実施中：黄色背景
3. レビュー中：薄ピンク背景
4. 期限切れ／着手遅れ：赤色背景 + 黄色文字

また、グループ化表示することで着手中／レビュー中のタスク、期限切れ／着手遅れのタスクを並び替えて表示できます。
これがすごい便利。
